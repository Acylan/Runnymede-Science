Number-Guessing Game

Write your first real program.

Build a short program that picks a secret number and tells you whether your guess is too high or too low until you win.

  • A computer
  • Scratch, Python, or an online editor like replit
  • Paper to plan the logic
  1. Plan the logic: pick a random number, ask for a guess, compare, repeat.
  2. In Python, use random.randint(1, 100) for the secret number.
  3. Use a while loop to keep asking until the guess is right.
  4. Use if / elif / else to say 'too high', 'too low' or 'correct!'.
  5. Count and display how many guesses it took.

This introduces the core ideas of computer science: variables store the secret number, a loop repeats the question, and conditionals make decisions. If you always guess the middle of the range you're doing a binary search — doubling the range only adds one extra guess, which is why it's so efficient.

No hazards — just remember to save your work.

Crack the Caesar Cipher

Send secret messages, then break them.

Encode messages by shifting the alphabet, then learn how codebreakers crack the code without the key.

  • Paper and pen
  • Two alphabet strips (a cipher wheel)
  • Optional: a computer
  1. Choose a shift, e.g. +3 so A→D, B→E, C→F.
  2. Encode a message by shifting every letter.
  3. Swap with a friend and decode using the reverse shift.
  4. Try to break a message without the key by testing all 25 shifts.
  5. Optional: write a program to do it instantly.

This is cryptography. A Caesar cipher has only 25 possible keys, so a computer cracks it by brute force in a fraction of a second. Frequency analysis helps too — 'E' is the most common letter in English, so spotting it reveals the shift. It shows why modern encryption needs astronomically many possible keys.

No hazards.

Messages your friends can read but eavesdroppers can't — until they apply the maths.

Make a Thaumatrope

Discover the trick behind every screen.

Build a spinning two-sided card that fuses two separate pictures into one — the illusion behind film and video.

  • Card or cardboard disc
  • String (or a pencil)
  • Markers
  • Scissors
  1. Draw a bird on one side and an empty cage on the other (flip it so they line up).
  2. Make two holes and thread string through, or tape the disc to a pencil.
  3. Spin it fast by twisting the string.
  4. Watch the bird appear inside the cage.
  5. Make your own pairs, or build a flip-book for more frames.

Because our eyes and brain blend rapid images together, two pictures shown fast enough merge into one. Films and screens use the same trick, showing 24–60 still frames every second so motion looks smooth. The number of images per second is called the frame rate.

Take care with scissors.

Count to 31 on One Hand

Think like a computer in binary.

Use your fingers as a computer's switches to count far past five and write messages in binary.

  • Your hands
  • Paper
  • Optional: a binary chart
  1. Give each finger a value: thumb 1, index 2, middle 4, ring 8, little 16.
  2. A raised finger means 1 (on); a lowered finger means 0 (off).
  3. Add up the raised fingers to make any number from 0 to 31.
  4. Write each number as 5 binary digits, e.g. 13 = 01101.
  5. Encode a letter using its number code.

Computers store everything as binary — bits that are either on (1) or off (0). Each place value doubles: 1, 2, 4, 8, 16. With n bits you can make 2ⁿ different combinations, which is why five fingers reach 31. Text is stored by giving every character its own binary code (ASCII or Unicode).

No hazards.

You can count to 31 on five fingers and write messages as 1s and 0s.
